Graph database model

Memory as connected contracts

Tailwag stores durable assistant memory as explainable Neo4j records with clear caller-owned IDs, evidence links, and adapter boundaries.

Core nodes Person, Episode, Event, Place, MemoryItem, directory rows, and biometric references cover the current scope.
Traceable links Relationships preserve participation, mentions, locations, directory matches, biometric references, and memory evidence.
Focused scope Neo4j is the only persistence layer; deferred ontology, confidence, org_id, and secondary stores stay out of scope.

Architecture is the source of truth: this deck summarizes the graph while the docs define every node, relationship, index, and deferred concept.

Person context and API surface

Context at the moment of use

Tailwag turns graph memory into prompt-ready and structured person context, then exposes selected package operations through an optional FastAPI adapter.

Durable recall Pulls active preferences, boundaries, facts, pets, follow-ups, directory profile lines, and recent target-person transcript lines.
HTTP adapter Routes live under /argos/providers/memory/resources/memory/request and require bearer auth except /health.
Embedding-only biometrics FastAPI biometric routes accept vectors for search, enrollment, observation, and voice-reference checks; raw media fields are rejected.

Sources, identity, and inspection

From activity to durable recall

Source adapters and operator tools feed the same memory contracts without adding source-specific graph labels or write-side inspection behavior.

Slack adapter Channels map to virtual places, threads become episodes, speakers participate, and mention-only users get MENTIONED_IN.
Directory sync Snowflake or local JSON rows become EmployeeDirectoryRecord nodes and support identity/profile projection.
Read-only inspection Follow-up validity, affect, person timeline, and memory item reports export HTML/JSON without changing Neo4j records.
Preference Durable person choices and interaction style.
Boundary Things to avoid, respect, or handle carefully.
Fact Narrow person-useful facts, not directory or ontology claims.
Follow-up Time-bounded open loops surfaced only while useful.
Biometrics Caller-supplied face and voice vectors on reference nodes.
Inspect Static reports for operators, not a runtime API surface.
Source Slack, live conversation, event, or directory evidence enters through approved inputs.
Store Neo4j records preserve identities, places, references, mentions, and evidence links.
